= Archetypes — template-only overlay data for `just init`

An *archetype* specialises the spine for one kind of repository (a Julia
library, a WordPress plugin, …) **as data, not as a fork**. Each archetype is
a directory here:

    archetypes/<name>/ARCHETYPE.a2ml     required — the contract (see below)
    archetypes/<name>/overlay/           optional — files copied over the spine

Decision record: `docs/decisions/0003-forge-and-sustain-lifecycle.adoc`.
Normative lifecycle: standards `spec/SCAFFOLD-LIFECYCLE.adoc`.

== The contract (`ARCHETYPE.a2ml`)

Single-line values, grep-parseable (same discipline as the canon's
`template-capability-gates.toml`):

- `[metadata]` — `archetype`, `schema_version`.
- `[capability] preset = "<name>"` — MUST name a preset in standards
  `.machine_readable/template-capability-gates.toml [presets]`. The archetype
  directory name and the preset name are the same string by convention.
- `[tokens]` — `TOKEN = "value"` lines. Only tokens the archetype can
  honestly determine (`PROJECT_KIND`, `PROJECT_DOMAIN`, `LANG_STACK`,
  `BUILD_CMD`, `TEST_CMD`, `DEPS`, `BUILD_OUTPUT_PATH`, `MAIN_FUNCTION`).
  Values MUST NOT contain `|` (the init sed delimiter — init skips such
  values with a warning). Judgement-only tokens
  (`PROJECT_UNIQUE_STRENGTH`, `TARGET_AUDIENCE`, `MUST_INVARIANTS`,
  per-repo names like `PACKAGE_NAME`) MUST be omitted so they stay
  `UNASSIGNED` — an archetype never writes fiction.

`just init <name>` applies the overlay directory (if present), adds the
token substitutions, and writes the preset into the minted repo's
`.machine_readable/rsr-profile.a2ml`. This whole `archetypes/` directory is
template-only and is removed from the minted tree, exactly like
`build/templates/`.
