This file maps every claim this repo's documentation makes to the real file
and line that backs it, with the caveats made explicit.

== "It's one self-contained file, no dependencies"

Claim: the entire game is `688-attack-hub.html`, playable by opening it
directly, no server or install required.

How this is checked:

* `ci.yml`'s `integrity` job asserts the file exists and is well-formed HTML
  on every push.
* `package.json` has no `dependencies` — only a `serve` convenience script
  wrapping `python3 -m http.server`, which is optional, not required.

Caveat: none known. This is the simplest claim in the repo and the easiest to
falsify — open the file in any browser.

== "It makes no network or exec calls"

Claim: the game is a purely client-side simulation. No `fetch`, no
`XMLHttpRequest`, no `WebSocket`, no `eval`, no real target host or exploit
primitive anywhere in the file.

How this is checked:

* `ci.yml`'s `integrity` job greps for `fetch(`, `XMLHttpRequest`,
  `WebSocket`, `eval(`, and `new Function(` on every push and fails the build
  if any appear.
* This was independently verified by direct inspection before the repo was
  created (see `AUDIT.adoc`).

Caveat: the grep is a pattern match, not a formal proof — see `AUDIT.adoc`
for exactly what that does and doesn't rule out.

== "It's not a copy of the 688(I) Hunter/Killer commercial game"

Claim: this is original fictional content coincidentally sharing the "688"
name (the game's fictional AS number, AS688), not a ROM or port of the 1997
submarine simulator.

How this is checked: manual inspection at repo creation — no commercial
game's copyright strings, asset names, or code appear anywhere in the file.
This is not re-checked by CI; it's a one-time provenance claim, recorded here
and in `AFFIRMATION.adoc`.

Caveat: if you're relying on this claim for a legal decision, don't take a
CLAUDE.md's word for it — read the file yourself. It's 39KB and self-contained.

== "There's no formal verification ledger (unlike f19-stealth-glider)"

Claim: this repo deliberately does **not** carry a `VERIFICATION.md`
claims-ledger, `src/verify*.mjs` scripts, or any "proven" language, unlike its
sibling `metadatastician/f19-stealth-glider`.

How this is implemented: there is no `VERIFICATION.md` file, and `AUDIT.adoc`
says so explicitly rather than manufacturing claims to fill the slot.

Caveat: this is a design choice, not an oversight — f19-stealth-glider has a
frozen level geometry with a provable no-go lemma to verify; this game has no
comparable formal structure to prove anything about. Inventing one would be
worse than having none.
