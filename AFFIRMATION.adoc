== What we affirm

=== The game file is what it claims to be

`688-attack-hub.html` is 39,156 bytes of self-contained HTML/CSS/JS (no
external `<script src>`, no `<link>` to a remote stylesheet, no `fetch`,
`XMLHttpRequest`, `WebSocket`, `eval`, or `new Function`). This was checked by
direct `grep` against the file before it was ever committed to this
repository, and is re-checked on every push by `ci.yml`.

policy), and its `rsr-profile.a2ml` capability-declaration approach were all

== Addendum — the orthographic globe

Anchored separately because it is *not* on `main` at the time of writing: it
lives on branch `feat/orthographic-globe`, commit `78b520b`, as an open draft
pull request. The affirmation above still describes the initial import
accurately; this section describes only what was added on that branch, and
should be folded into the anchor above if and when it merges.

What was measured, in a real browser (Chrome, one machine, one session),
against the running game rather than the source:

* Trans-Pacific backbone runs (`sea|nrt`, `sjc|nrt`, `sjc|syd`) each emit
  *two* path subpaths, at x≈1 and x≈951 — that is, they leave one edge of
  the map and re-enter the other. Short runs (`lhr|ams`) and trans-Atlantic
  runs (`iad|lhr`) emit one. This is the specific check that catches a
  longitude lerp drawing the Pacific cable across Eurasia while leaving both
  endpoints correct.
* Of 20 regions, the 6 then on the far side all reported
  `pointer-events: none`, and none of the 14 near-side ones did.
  `document.elementFromPoint()` at a hidden node's own coordinates returned
  the land polygon, not the node — you cannot click through the planet.
* One `positionMap()` pass cost 0.34 ms against a 16.7 ms frame budget,
  across 181 SVG elements.
* The page loaded and ran with no console errors, and the file still contains
  no `fetch(`, `XMLHttpRequest`, `WebSocket`, `eval(` or `new Function(`.
* Both projections were seen rendering: the flat map at game start, and the
  globe mid-run with great-circle arcs, limb-clipped coastlines, and a
  working tooltip.

What this addendum does *not* affirm:

Correctness of the projection.:: Four properties were measured. That is not
the same as the globe being cartographically right, and there is no automated
test for any of it — every check above was run once, by hand, and would not
catch a regression.

Anything about other browsers or devices.:: One browser, one machine. No
mobile, no low-end hardware, no cross-browser pass.

That the coastline limb seam is correct.:: It is knowingly wrong — polygons
straddling the horizon close along a chord, not the limb arc. See
`ARCHITECTURE.md`.
