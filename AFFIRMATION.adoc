== What we affirm

=== The game file is what it claims to be

`688-attack-hub.html` is 39,156 bytes of self-contained HTML/CSS/JS (no
external `<script src>`, no `<link>` to a remote stylesheet, no `fetch`,
`XMLHttpRequest`, `WebSocket`, `eval`, or `new Function`). This was checked by
direct `grep` against the file before it was ever committed to this
repository, and is re-checked on every push by `ci.yml`.

=== The repository structure mirrors a real, working precedent

`metadatastician/f19-stealth-glider` was read directly (`gh api ... /contents`,
file-by-file) as the structural precedent for this repo: its pruned file set,
its `pages.yml` hand-packed-artifact pattern (required because
`actions/upload-pages-artifact` is unusable under this org's SHA-pinning
policy), and its `rsr-profile.a2ml` capability-declaration approach were all
confirmed present and green (its CI, Pages deployment, and repository
visibility were checked live, not recalled from memory) before being adapted
here.

=== The repo was scaffolded from the real GitHub template, not a stale local copy

`hyperpolymath/rsr-template-repo` was confirmed live via `gh repo view`
(`isTemplate: true`, `visibility: PUBLIC`, pushed 2026-07-21) before
`gh repo create --template` was run. The known-stale local checkout of this
template was deliberately not used as a source.

== What we do NOT affirm

Stated because an affirmation that only lists successes is marketing.

No formal verification ledger.:: Unlike `f19-stealth-glider`, this repo makes
no provable design claims and ships no `VERIFICATION.md` claims ledger. See
`AUDIT.adoc` for why.

No accessibility, performance, or balance testing.:: The game has not been
audited for WCAG conformance, profiled for performance, or playtested for
balance beyond the author's own judgement.

The "original content, not a commercial-game copy" claim is a one-time manual
read, not a mechanised check.:: Nothing in CI re-verifies provenance on every
push; there is nothing meaningful to grep for that would strengthen this
beyond the initial inspection.

The Globe.gl / engine-migration roadmap in `ARCHITECTURE.md` is aspiration,
not work in progress.:: Neither has been started. Treat that section as
options under consideration, not a commitment.
